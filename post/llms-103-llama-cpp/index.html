llama.cpp khá nổi tiếng và gặp rất nhiều trong các blog về triển khai AI, bản thân nó cũng gây nhiều nhiều nhầm lẫn về vai trò, chức năng. Mục tiêu ở đây là hiểu bản chất của llama.cpp và hệ sinh thái xung quanh nó:

  • Repo llama.cpp được mô tả là LLM inference in C/C++, tức là một thư viện nguồn mở (OSS library) code bằng pure C/C++, không phụ thuộc vào bất cứ thư viện nào bên ngoài.
  • Georgi Gerganov phát triển llama.cpp với mục tiêu ban đầu là có thể chạy LLaMA (một họ model do Meta AI phát hành) trên các máy tính với phần cứng thấp mà không cần GPU đắt đỏ. Nhưng sau này llama.cpp cũng đã hỗ trợ nhiều backends khác nhau từ CPU, GPU, NPU … và các họ model khác như Gemma, Grok, GLM, Mistral …
  • Không chỉ là thư viện, repo cũng cung cấp vài tools để có thể chạy một model ở dạng local, quản lý, tải, convert model, cũng như cung cấp local model dưới dạng LLM Inference.
  • llama-cli: một CLI tool, kéo một model từ HF và bật một conversation mode để tương tác với model, ví dụ với gemma-3-1b-it-GGUF:
    • Tải model từ HuggingFace (lưu ý là format GGUF)
    • Khởi động một conversation mode để chat, nhận response (param -m model.gguf)
    • Có thể custom chat template, custom các config về CPU/GPU/Memory …

1. GGUF

  • GGUF (GPT-Generated Unified Format) là một định dạng file dạng nhị phân (tương tự các tệp .zip, .csv), dùng để lưu trữ model và được thiết kế để tối ưu cho việc loading/saving nhanh, giúp hiệu quả cho mục đích suy luận. GGUF được thiết kế để sử dụng với GGML và các executors based on GGML. GGUF được phát triển bởi @ggerganov, cũng là tác giả của llama.cpp một C/C++ LLM inference framework rất nổi tiếng khác. Các model được phát triển bởi các framework như PyTorrch có thể convert sang GGUF forrmat để chạy.
  • GGUF Naming Convention: <BaseName><SizeLabel><FineTune><Version><Encoding><Type><Shard>.gguf. Trong đó SizeLabel: Parameter weight class (useful for leader boards) represented as <expertCount>x<count><scale-prefix>
    • Q: Quadrillion parameters.
    • T: Trillion parameters.
    • B: Billion parameters.
    • M: Million parameters.
    • K: Thousand parameters.
  • Ví dụ về GGUF naming:

    • Mixtral-8x7B-v0.1-KQ2.gguf:

      • Model Name: Mixtral
      • Expert Count: 8
      • Parameter Count: 7B
      • Version Number: v0.1
      • Weight Encoding Scheme: KQ2

    • Grok-100B-v1.0-Q4_0-00003-of-00009.gguf

      • Model Name: Grok
      • Expert Count: 0
      • Parameter Count: 100B
      • Version Number: v1.0
      • Weight Encoding Scheme: Q4_0
      • Shard: 3 out of 9 total shards
  • Key Benefits of GGUF:
    • Chạy AI model trên CPU mà ko cần GPUs
    • Quantization giảm kích thước model từ 50-70%
    • Một file duy nhất
    • Chạy được trên nhiều OS như Windows, Mac, Linux
    • Hỗ trợ bởi nhiều major local AI
