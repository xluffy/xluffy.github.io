13/05/2026 - How to persistent UTM tracking in GTM

2026-05-13
#UTM  #tracking  #GTM 

1. UTM là gì?

UTM là từ viết tắt của Urchin Traffic Monitor, được phát minh bởi một công ty tên là Urchin, sau đó cty này được Google mua lại vào năm 2005 để bổ trợ cho analytics platform của Google lúc đó là Google Analytics. Kết quả là UTM trở thành tiêu chuẩn của ngành, cách thức phổ biến để theo dõi các traffic source và distribution channels.

Bài toán: Chúng ta có một website, nhu cầu là muốn biết user tới website thông qua các kênh nào. Từ kết quả đó, chúng ta biết được kênh nào là hiệu quả, để tập trung vào kênh đó. Xa hơn nữa với các kênh quảng cáo trả tiền, ta biết được hiệu quả để có cần duy trì hay tăng cường chi phí hay không. Nâng cao hơn nữa, kết hợp với doanh thu, chúng ta biết hiệu quả của quảng cáo so với doanh thu, lợi nhuận để đánh giá một chiến dịch marketing có hiệu quả hay không.

Có rất nhiêu các kênh phân phối traffic như:

  • Organic - SEO
  • Google Ads - Paid
  • Facebooks Ads - Paid
  • Influencer - partnership campaigns
  • Affiliate link
  • Blog

Có các UTM parameter như sau:

  • utm_id: Campaign ID
  • utm_source: Referrer ví dụ google, facebook, newletter, tiktok, email … -> Giúp ta biết user tới website từ nguồn nào
  • utm_medium: Marketing medium ví dụ như cpc (cost-per-click), paid_social, display …
  • utm_campaign: product, sologan, promo-code ví dụ spring_2026_sale, summer_2025_sale … -> Biết campain mà user tham gia
  • utm_term: Paid keyword ví dụ như shoes, running+gear
  • utm_content: Use to differentiate creatives. For example, if you have two call-to-action links within the same email message, you can use utm_content and set different values for each so you can tell which version is more effective.

Sau đó có thể dùng công cụ như Campaign URL builder để build URL, ví dụ:

https://taoquangne.com?utm_id=&utm_source=facebook&utm_medium=cpc&utm_campaign=summer_sale

Khi post link này lên facebook hoặc chạy quảng cáo FB, user click vào link, trên Google Analytic sẽ có dữ liệu đo lường để biết user tới từ đâu.

Tóm lại:

  • Web site cần gắn GTM để dễ dàng deploy các tracking như GA
  • Tạo URL builder với các UTM parameters để tracking
  • Phân phối URL qua các kênh marketing (paid, organic …)
  • Truy cập Google Analytic theo dõi số liệu và đánh giá kết quả

2. Deploy tracking code

Để tracking được các metrics trên website, chúng ta cần các công cụ để thu thập dữ liệu (Google Analytics, HotJar, Mixpanel, Heap …), để các công cụ thu thập được dữ liệu, chúng ta cần deploy lên website. Thông thường, trước đây để có thể deploy tracking code, Marketer sẽ gửi đoạn code tracking được gen bởi các Tool và nhờ Developer gắn vào Frontend, thông thường gắn vào thẻ <head> của HTML dạng như sau:

3. Persistent

Parameter trên URL sẽ biến mất khi user navigate giữa các page. Ví dụ nếu user click vào link có UTM, sau đó click sang home-page, sang product page … giá trị UTM trên URL sẽ mất và lúc đó việc tracking sẽ không chính xác.

Vấn đề càng trầm trọng hơn khi chiến dịch marketing là chiến dịch trả tiền và marketer cần đo lường tỉ lệ chuyển đổi từ chi phí ra đơn hàng.

Ví dụ: Mong muốn user sau khi click vào Google Ads -> dẫn vào trang cammpaign -> Add sản phẩm vào cart -> Thanh toán -> Ra đơn.

Khi user di chuyển giữa các trang, do parameter bị clear, nên chỉ ghi nhận vào trang campaign, và khi user thanh toán, GA ghi nhận đó là “Direct traffic” -> khiến việc đánh giá hiệu quản chiến dịch dựa trên chi phí sẽ sai.

Để giải quyết bài toán này, chúng ta cần “tóm” các tham số UTM và lưu vào đâu đó và khi cần dùng thì lấy lên lại. Cơ bản có 2 hướng:

  • Lưu trữ phía trình duyệt - client - Lưu phía trình duyệt cũng có 2 dạng là:
    • Cookie: Cookie có thể truy cập ở client, và có thể attach vào header Cookie gửi lên server. Cookie có expires, hết hạn sẽ mất, cookie chỉ cho lưu tối đa 4KB
    • LocalStorage chỉ có thể truy cập ở client, có dung lượng tối đa 10MB và chỉ bị clear khi user xóa cache
  • Lưu trữ phía server side - cách này ổn nhất nhưng chỉ phù hợp nếu là owner của source code, cơ chế là khi user truy cập vào URL, Backend bóc tách, đọc UTM parameter và lưu vào session của người dùng (Database hoặc Redis), khi phát sinh chuyển đổi (checkout/payment) thì lấy lên để ghi nhận.

Cả 2 cách trên, phù hợp cho developer, mặc dù với cách lưu ở phía client có thể treat để deploy qua GTM mà không cần phải là developer.

Ngoài ra, có một cách khác nữa phù hợp với marketer đó là dùng các Custom Template như “Persist UTM” hoặc “Simo Ahava’s Cookie Tool”, deploy qua GTM, tự động quét URL, lưu UTM vào cookie và đọc ngược lại, bản chất cũng như việc lưu ở client, nhưng không cần Developer hỗ trợ deploy.

3. Triển khai

Bài viết dành cho marketer, nên chỉ tập trung vào các giải pháp có thể triển khai thông qua GTM

<script>
  (() => {
    const KEYS = [
      "utm_source",
      "utm_medium",
      "utm_campaign",
      "utm_term",
      "utm_content",
    ];

    const COOKIE_DAYS = 90;
    const ATTRIBUTION_MODE = "first-touch";
    // "first-touch" | "last-touch"

    function getUrlParams() {
      const params = new URLSearchParams(window.location.search);
	  const result = {};
	  
      KEYS.forEach((key) => {
        const value = params.get(key);

        if (value !== null) {
          result[key] = value;
        }
      });

      return result;
    }

    function getCookies() {
      return document.cookie
        .split("; ")
        .reduce((acc, cookie) => {
          const [name, value] = cookie.split("=");

          acc[name] = decodeURIComponent(value || "");

          return acc;
        }, {});
    }

    function setCookie(name, value, days) {
      const expires = new Date(
        Date.now() + days * 86400000
      ).toUTCString();

      document.cookie = [
        `${name}=${encodeURIComponent(value)}`,
        `expires=${expires}`,
        "path=/",
        "SameSite=Lax",
        "Secure",
      ].join("; ");
    }

    const incomingParams = getUrlParams();
    const existingCookies = getCookies();

    KEYS.forEach((key) => {
      const hasExisting = Boolean(existingCookies[key]);
      const hasIncoming = key in incomingParams;

      if (!hasIncoming) {
        return;
      }

      const shouldWrite =
        ATTRIBUTION_MODE === "last-touch" || !hasExisting;

      if (shouldWrite) {
        setCookie(key, incomingParams[key], COOKIE_DAYS);
      }
    });
  })();
</script>
